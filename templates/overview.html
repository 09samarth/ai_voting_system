Project overview

Architecture, accessibility, and ethical scope of the AI Voice Voting System demo.

Home Voice voting Results

System architecture

  • Web layer (Flask): Serves separate pages for voice voting, results, and documentation, plus JSON APIs under /api/*.
  • Voice layer: A dedicated Python subprocess captures speech, normalizes numeric voter IDs, and records votes.
  • Data layer (SQLite): Stores synthetic voter IDs, demo candidates, and vote records locally for experimentation.

Accessibility and usability

  • Clear voice prompts and status messages at each step of the voting flow.
  • Keyboard-friendly, responsive layout with large touch targets.
  • Dedicated pages help users focus on one task at a time instead of a cluttered single screen.

Ethical, academic-only demo

This system is intentionally limited to non-sensitive, synthetic data:

  • Numeric state-coded voter IDs are randomly generated for testing only.
  • No real-world voter registries, identities, or personal details are used.
  • The code is suitable for college or research demonstrations of speech interfaces and E2E flows.
